Спецификация архитектуры шлюза

Техническое руководство по интеграции On-Premise прокси-шлюза Braint в изолированные контуры корпоративных систем. Архитектура построена на базе принципов нулевого доверия (Zero-Trust).

1.1. Локальное развертывание

Шлюз поставляется в виде скомпилированного Docker-манифеста и разворачивается во внутреннем периметре организации без необходимости предоставления интернет-доступа к хост-системе.

Terminal (Bash) 
# Клонирование стабильного релиза в закрытый контур
git clone https://github.com/braint-ai/gateway-stable.git && cd gateway-stable

# Конфигурация изолированных переменных окружения
cat <<EOF > .env
GATEWAY_PORT=8000
TRANSFORMERS_OFFLINE=1
REDIS_TTL_DAYS=30
EOF

# Запуск оркестрированного стека контейнеров
docker-compose up -d --build

1.2. Air-Gap изоляция

Параметр среды Значение Регламент ИБ безопасности
TRANSFORMERS_OFFLINE `1` Блокирует сетевой доступ локальной SLM-модели наружу. Веса считываются строго из локального Docker Volume.
REDIS_TTL_DAYS `30` Время жизни временной таблицы де-токенизации в RAM-памяти перед безвозвратным удалением сессии.

2.1. Единый метод безопасного обмена

Для интеграции с внутренними CRM/ERP системами замените базовый URL провайдера ИИ на адрес локального шлюза. Спецификация полностью обратно совместима.

POST http://localhost:8000/api/chat
Входящий JSON запроса: 
{
  "message": "Контрагент ТОО Квант, ИИН 790807300123",
  "session_id": "session_corp_451"
}
Что уходит наружу в LLM-сеть: 
{
  "message": "Контрагент [COMPANY_1], ИИН [ID_IIN_1]",
  "session_id": "session_corp_451"
}

3.1. Морфологический синтез агглютинативных суффиксов

В отличие от стандартных Regex-систем, шлюз Braint интегрирует модуль сопоставления падежей и законов сингармонизма казахского языка. Если внешняя модель возвращает токенизированную форму с суффиксом падежа (например, [COMPANY_1]-ке бағытталды), шлюз автоматически рассчитывает фономорфологию корня компании при обратной подстановке, превращая текст в грамматически верный вариант: «Қазақмысқа бағытталды».

3.2. Регламент безопасного In-Memory хранения данных

Для исключения возможности компрометации данных при физическом извлечении дисков, соответствия исходных данных и сгенерированных токенов удерживаются исключительно в энергозависимой RAM-памяти инсталляции (In-Memory Isolation). Логирование событий для внешних SIEM-систем безопасности фиксирует факт транзакции, но исключает запись конфиденциальных промптов.
