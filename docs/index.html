Спецификация архитектуры и API

Braint AI Gateway функционирует как прозрачный комплаенс-прокси между вашей внутренней корпоративной инфраструктурой и глобальными моделями ИИ. Архитектура разработана по стандартам Zero-Trust.

1.1. Развертывание On-Premise шлюза

Шлюз поставляется в виде изолированных Docker-контейнеров и разворачивается внутри демилитаризованной зоны (DMZ) или закрытого контура организации. Система оптимизирована для инференса на CPU.

Terminal (Bash) Docker Setup 
# Шаг 1: Клонируйте стабильный дистрибутив в закрытый контур
git clone https://github.com/braint-ai/gateway-stable.git && cd gateway-stable

# Шаг 2: Создайте файл конфигурации окружения
cat <<EOF > .env
GATEWAY_PORT=8000
EXTERNAL_LLM_PROVIDER=openrouter
EXTERNAL_LLM_API_KEY=your_secured_enterprise_key_here
REDIS_TTL_DAYS=30
TRANSFORMERS_OFFLINE=1
EOF

# Шаг 3: Запустите локальный оркестрованный стек одной командой
docker-compose up -d --build

1.2. Параметры изоляции (Air-Gap Комплаенс)

Для полного исключения несанкционированных сетевых запросов шлюз поддерживает режим жесткой изоляции. Конфигурация переменных среды в `.env` файле:

Переменная Значение Описание политики ИБ
TRANSFORMERS_OFFLINE `1` Намертво блокирует обращение локальной SLM-модели (NER) к внешним репозиториям HuggingFace. Веса загружаются из локального тома Docker.
REDIS_TTL_DAYS `30` Время жизни связки «Токен <-> Персональные данные» в оперативной памяти. По истечении срока данные безвозвратно удаляются.

2.1. Метод безопасного обмена сообщениями

Замените базовый эндпоинт провайдера во всех ваших CRM, ERP и чат-системах на адрес локального шлюза Braint. Интерфейс полностью идентичен стандартным спецификациям обмена сообщениями.

POST http://localhost:8000/api/chat

Пример входящего запроса (JSON):

{
  "message": "Срочно проверь благонадежность контрагента. ТОО Квант, БИН 010203040506, директор Смагулов А.Б., ИИН 790807300123.",
  "session_id": "corp_session_9941"
}

Что уходит во внешнюю LLM-сеть:

{
  "message": "Срочно проверь благонадежность контрагента. [COMPANY_1], БИН [ID_BIN_1], директор [PERSON_1], ИИН [ID_IIN_1].",
  "session_id": "corp_session_9941"
}

3.1. Двуязычный морфологический синтез

Главная проблема простых систем анонимизации при работе с флективными (русский) и агглютинативными (казахский) языками — это грамматическая деградация при де-токенизации. Шлюз Braint интегрирует гибридный пайплайн **Morphological Synthesis Layer** на базе конечных автоматов и локальных SLM.

Пример агглютинации в казахском языке:
Если исходное название компании «Қазақмыс» маскируется токеном, а внешняя модель возвращает синтаксическую конструкцию с суффиксом падежа: `[COMPANY_1]-ке бағытталды`.
Шлюз автоматически отсекает несовместимый суффикс `-ке`, анализирует фономорфологию восстановленного корня по закону сингармонизма и синтезирует грамматически безупречный ответ: «Қазақмысқа бағытталды» вместо сломанного «Қазақмыс-ке...».

3.2. Архитектура хранения таблиц соответствия (Redis Vault)

Де-токенизация требует сопоставления сгенерированных токенов с исходным контекстом. В целях безопасности шлюз полностью отказывается от сохранения этих связок на энергонезависимые жесткие диски (SSD/HDD).

✔

**In-Memory изоляция**: Все пары `Токен <-> Исходный текст` хранятся исключительно во внутренней резидентной базе данных Redis, развернутой внутри RAM-памяти сервера.

✔

**Автономное вытеснение (TTL)**: Каждая сессия жестко привязана к глобальному таймеру автоматического уничтожения. Данные стираются без возможности восстановления.

3.3. Режим аудита (Audit Trail) для офицеров ИБ

Система фиксирует факт прохождения транзакций, фиксирует тип обнаруженных сущностей (ИИН, ФИО, Локальный БИН), но **не сохраняет конфиденциальный текст в лог-файлы общего доступа**. Это позволяет интегрировать шлюз с корпоративными SIEM-системами банка без нарушения комплаенса.
